Having No Data….You Do Not Need Application To Manage It BY PLTOD ON NOVEMBER 16, 2011 Have you thought that “data” is the central concept in your applications? Basically, we need applications to manage data. Lets leverage this and structure our JavaScript concept explanation around it. Imagine you have a data life cycle process which has the following stages: Stage 1: Data Creation – Introduce data into our applications Stage 2: Data Resolution – Resolve data so we can use it in the next two stages Stage 3: Data Update – Resolved data is updated Stage 4: Data Deletion – Resolved data is deleted So how we can create data? It is as simple as this: Variable declarations and function declarations are the only way we could introduce data into our applications! For example if we have: var v = 5; function f (p) {return p;} For each variable declaration, function declaration, and its function formal parameter JS introduces data into our applications and creates identifier that will be used for accessing it! In our case the identifiers are v, f and p. The corresponding data is 5, function itself and the value passed to p on function invocation. Wait what about objects? Aren’t they a mean for introducing new data? In short they are not. We create objects like this: var o = {propO: “value”} …which is still variable declaration, so we have identifier = o and data = the associated object. I know information here is kind of out of context. But wait until my next post regarding the next stages of data lifecycle and you will get the complete picture.
